# Kubernetes Cluster Setup

## Cluster Overview

This document describes the setup of a local Kubernetes cluster running on Proxmox VMs.

## Node Inventory

| Role | Hostname | IP Address | vCPU | RAM |
|---|---|---|---|---|
| Control Plane | k8s-cp-01 | 10.10.10.201 | 2 | 4 GB |
| Worker | k8s-worker-01 | 10.10.10.202 | 2 | 4 GB |
| Worker | k8s-worker-02 | 10.10.10.203 | 2 | 4 GB |

## Kubernetes Components

- Container runtime: containerd

- Cluster bootstrap tool: kubeadm

- Node agent: kubelet

- CLI: kubectl

- CNI: Flannel

- Pod CIDR: 10.244.0.0/16

## Hostname Configuration

Each VM was assigned a hostname:

```bash
sudo hostnamectl set-hostname k8s-cp-01
sudo hostnamectl set-hostname k8s-worker-01
sudo hostnamectl set-hostname k8s-worker-02
```

## Hosts File Entries

The following entries were added to /etc/hosts on all nodes:

```
10.10.10.201 k8s-cp-01
10.10.10.202 k8s-worker-01
10.10.10.203 k8s-worker-02
```

## Swap Disabled

Swap was disabled because Kubernetes expects swap to be off by default:

```bash
sudo swapoff -a
sudo sed -i '/ swap / s/^/#/' /etc/fstab
```

## Kernel Modules

Required modules were configured:

```bash
sudo tee /etc/modules-load.d/k8s.conf > /dev/null <<EOF
overlay
br_netfilter
EOF

sudo modprobe overlay
sudo modprobe br_netfilter
```

## Sysctl Networking Settings

Kubernetes networking settings were configured:

```bash
sudo tee /etc/sysctl.d/k8s.conf > /dev/null <<EOF
net.bridge.bridge-nf-call-iptables  = 1
net.bridge.bridge-nf-call-ip6tables = 1
net.ipv4.ip_forward                 = 1
EOF

sudo sysctl --system
```

## Container Runtime

Installed and configured `containerd`:

```bash
sudo apt update
sudo apt install -y containerd

sudo mkdir -p /etc/containerd
containerd config default | sudo tee /etc/containerd/config.toml > /dev/null

sudo sed -i 's/SystemdCgroup = false/SystemdCgroup = true/' /etc/containerd/config.toml

sudo systemctl restart containerd
sudo systemctl enable containerd
```

## Kubernetes Package Installation

Installed Kubernetes packages from the official Kubernetes package repository:

```bash
sudo apt update
sudo apt install -y apt-transport-https ca-certificates curl gpg

sudo mkdir -p /etc/apt/keyrings

curl -fsSL https://pkgs.k8s.io/core:/stable:/v1.30/deb/Release.key \
  | sudo gpg --dearmor -o /etc/apt/keyrings/kubernetes-apt-keyring.gpg

echo 'deb [signed-by=/etc/apt/keyrings/kubernetes-apt-keyring.gpg] https://pkgs.k8s.io/core:/stable:/v1.30/deb/ /' \
  | sudo tee /etc/apt/sources.list.d/kubernetes.list

sudo apt update
sudo apt install -y kubelet kubeadm kubectl
sudo apt-mark hold kubelet kubeadm kubectl
sudo systemctl enable kubelet
```

## Control Plane Initialization

The control plane was initialized on k8s-cp-01:

```bash
Copy
sudo kubeadm init \
  --apiserver-advertise-address=10.10.10.201 \
  --pod-network-cidr=10.244.0.0/16 \
  --node-name=k8s-cp-01
```

Configured kubectl access:

```bash
Copy
mkdir -p $HOME/.kube
sudo cp -i /etc/kubernetes/admin.conf $HOME/.kube/config
sudo chown $(id -u):$(id -g) $HOME/.kube/config
```

## CNI Installation

Installed Flannel:

```bash
kubectl apply -f https://github.com/flannel-io/flannel/releases/latest/download/kube-flannel.yml
```

## Worker Node Join

Worker nodes were joined using the command generated by `kubeadm init`.

If needed, a new join command can be generated from the control plane:

```bash
kubeadm token create --print-join-command
```

## Cluster Verification

Checked node status:

```bash
kubectl get nodes
kubectl get nodes -o wide
kubectl get pods -A
```
